# Demo测试数据交互规则

## 1. 目标
用于验证 Demo 的“多源取数 + 多Sheet计算 + 同Sheet公式 + 样式保留 + 合并单元格”链路。

## 2. 文件清单
目录：`C:\projects\epc-test\demo-data`

1. `demo_source_pl.xlsx`（sheet: `PL附表-2320、2355`）
2. `demo_source_vat_output.xlsx`（sheet: `增值税销项 `，注意尾部空格）
3. `demo_source_vat_input.xlsx`（sheet: `增值税进项认证清单`）
4. `demo_source_config.xlsx`（sheet: `公司代码配置表`）
5. `demo_ledger_template.xlsx`（sheet: `模板说明`、`参数区`、`Demo_台账输出`）

## 3. 主键与匹配规则

- 业务主键：`项目编码`
- 主表：`PL附表-2320、2355`
- 关联方式：
1. `PL附表-2320、2355.A列` = `增值税销项 .A列`
2. `PL附表-2320、2355.A列` = `增值税进项认证清单.A列`
3. `PL附表-2320、2355.A列` = `公司代码配置表.A列`

若任一来源缺失对应项目编码：
- 金额字段默认填 `0`
- 状态字段写 `MISSING_SOURCE`

## 4. 字段映射规则

目标sheet：`Demo_台账输出`，从第3行开始写数据。

1. `A列 项目编码` <- `PL附表-2320、2355.A列`
2. `B列 项目名称` <- `PL附表-2320、2355.B列`
3. `C列 PL金额` <- `PL附表-2320、2355.C列`
4. `D列 销项税额` <- `增值税销项 .B列`
5. `E列 进项税额` <- `增值税进项认证清单.B列`
6. `F列 应纳税额(同表公式)` = `D - E`（保留公式）
7. `G列 交叉校验(跨Sheet公式)` = `F - 参数区!$B$2`（保留跨sheet公式）

## 5. 公式与计算规则

- 行级公式：
1. `F3 = D3 - E3`
2. `G3 = F3 - 参数区!$B$2`
3. 下方行按相对引用递推

- 汇总行（第6行）公式：
1. `C6 = SUM(C3:C5)`
2. `D6 = SUM(D3:D5)`
3. `E6 = SUM(E3:E5)`
4. `F6 = SUM(F3:F5)`
5. `G6 = SUM(G3:G5)`

## 6. 样式与结构规则

1. `A1:G1` 必须合并（标题区）
2. 第2行为表头，使用统一表头样式（加粗 + 填充色）
3. 金额列（C~G）使用数值格式 `#,##0.00`
4. 禁止运行时重建颜色/边框，优先保留模板样式

## 7. 预期计算结果（用于断言）

基于当前样例数据：

1. `F3 = 10800 - 6200 = 4600`
2. `F4 = 7200 - 2800 = 4400`
3. `F5 = 4500 - 1800 = 2700`
4. `F6 = 11700`
5. `参数区!B2 = 2000`
6. `G3 = 2600`
7. `G4 = 2400`
8. `G5 = 700`
9. `G6 = 5700`

## 8. 失败判定规则

任一条件满足即判失败：

1. 任一必需sheet不存在（含名称不一致，例如尾部空格丢失）
2. 输出sheet未生成或标题合并单元格缺失
3. 关键公式被写成纯值（`F3~F6`, `G3~G6`）
4. 公式重算后出现 `#REF!`、`#DIV/0!`
5. 汇总值不等于明细求和

## 9. 推荐执行顺序

1. 先做读取与主键关联（不写公式）
2. 再写公式并重算
3. 最后校验样式、合并单元格、错误值
