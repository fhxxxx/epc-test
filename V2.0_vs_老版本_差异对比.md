## 三、睿景景程月结数据表-报税 模板详情

### 3.1 模板结构

| 列名 | 说明 | 示例 |
|------|------|------|
| 行标签 | 项目名称 | 正镶白旗10万风电治沙项目 |
| 项目代码 | 项目代码 | 4305 |
| 成本 | 成本金额 | (空) |
| 收入 | 收入金额 | (空) |
| 销项 | 销项税额 | (空) |
| 已开票收入 | 已开票收入 | (空) |
| 已开票税额 | 已开票税额 | (空) |
| 未开票收入 | 未开票收入 | (空) |

### 3.2 重要说明

- **第1行**: 取自：报税pt（筛选：月份）
- **第2行**: 核对：已开票收入+未开票收入=账面主营收入
- **第3行**: 建安（业务类型标识）

### 3.3 使用场景

- 仅2320、2355公司需要上传此文件
- 用于PL表附表中"建安区域"相关的数据取值
- 不参与台账生成，仅在PL附表计算时作为数据源

## 六、对设计文档的影响

### 6.1 需要更新的内容

1. **文件类型枚举**（第3.2.2节）
   - 新增：`RJ_MONTHLY_SETTLEMENT` - 睿景景程月结数据表-报税
   - 新增：`CONTRACT_STAMP_TAX` - 合同印花税明细台账

2. **Sheet页分类**（第5.3.1节）
   - 在"类型A：直取型"中更新Sheet页说明
   - 明确辅助Sheet不参与台账生成

3. **文件存储设计**（第8.1节）
   - 更新Azure Blob目录结构，包含新文件类型

4. **待确认事项**（第11章）
   - 更新睿景景程月结数据表状态为"✅ 已确认"
   - 新增合同印花税明细台账确认事项

### 6.2 不需要修改的内容

- 核心业务逻辑（增值税变动表、Summary表等）保持不变
- 权限模型、数据湖API等架构设计不变
- 技术栈和开发流程不受影响
