## 4. 填报与计算流程（先手工后计算）

1. 用户先手工维护两块区域的 `拆分依据(splitBasis)`：
- Section 1：常见值如 `13%`、`9%`、`6%`、`普票-6%`
- Section 2：常见值如 `专票-13%`、`专票-9%`、`专票-6%`、`普票-6%`
2. Section 1 与 Section 2 的 `splitBasis` 必须一一对应且税率口径一致（按税率维度校验一致性）。
3. 系统在读取到 `splitBasis` 后，按税率与票种规则计算其余金额列。
4. 若任一 `splitBasis` 为空、或无法提取税率（未包含如 `6%/9%/13%` 这类税率片段），必须校验拦截并报错，不允许继续计算。

### 4.1 拆分依据一一对应校验（答疑补充）

- 上下两张表必须按“税率口径”一一对应：
  - `13%` ↔ `专票-13%`
  - `9%` ↔ `专票-9%`
  - `6%` ↔ `专票-6%`
  - `普票-6%` ↔ `普票-6%`
- 若任一侧缺失对应项，或出现多条重复项，系统应报错并终止计算。
- 建议错误文案：`拆分依据不一致：未在[上/下]表找到与 {splitBasis} 对应的税率项`。

### 4.2 专票/普票处理分流（答疑补充）

- **专票场景**：按规则从 `睿景景程月结数据表-报税` 取数（系统计算）。
- **普票场景**：金额由用户手工填写，系统不自动计算。
- 普票行允许为空；若普票行相关金额列全部为空，则自动删除该行对应 `splitBasis`（视为未填报该口径）。

## 5. 剩余字段取值逻辑（除splitBasis外）

### 5.1 Section 1 - `uninvoicedIncome`（未开票收入）
- 公式：`uninvoicedIncome = declaredAmount - invoicedIncome`
- 其中 `declaredAmount` 来自 Section 2 同税率口径的 `declaredAmount`（按税率匹配，且要求对应 `splitBasis` 一致）。

### 5.2 Section 1 - `outputTax`（销项）
- 公式：`outputTax = declaredTaxAmount - invoicedOutputTax`
- 其中 `declaredTaxAmount` 来自 Section 2 同税率口径的 `declaredTaxAmount`（按税率匹配，且要求对应 `splitBasis` 一致）。

### 5.3 Section 1 - `invoicedIncome`（已开票收入）
- 从 `splitBasis` 提取税率（如 `专票-6% -> 6%`、`13% -> 13%`）。
- 在 `睿景景程月结数据表-报税` 按 `title contains 提取税率` 汇总 `invoicedIncome`，写入 `invoicedIncome`。

### 5.4 Section 1 - `invoicedOutputTax`（已开票销项）
- 从 `splitBasis` 提取税率。
- 在 `睿景景程月结数据表-报税` 按 `title contains 提取税率` 汇总 `invoicedTaxAmount`，写入 `invoicedOutputTax`。

### 5.5 Section 2 - `declaredAmount`（申报金额）
- 从 `splitBasis` 提取税率（如 `专票-13% -> 13%`）。
- 在 `睿景景程月结数据表-报税` 按 `title contains 提取税率` 汇总 `income`（收入）写入 `declaredAmount`。

### 5.6 Section 2 - `declaredTaxAmount`（申报税额）
- 从 `splitBasis` 提取税率。
- 在 `睿景景程月结数据表-报税` 按 `title contains 提取税率` 汇总 `outputTax`（销项）写入 `declaredTaxAmount`。

### 5.7 普票行填报逻辑（答疑补充）
- 当 `splitBasis` 为 `普票-*` 时：
  - Section 1 的 `uninvoicedIncome/outputTax/invoicedIncome/invoicedOutputTax` 由用户手工填写。
  - Section 2 的 `declaredAmount/declaredTaxAmount` 由用户手工填写。
  - 系统不从月结表覆盖这些值。
- 若同一普票行可填字段全部为空，则自动删除该行，不参与后续一一对应与汇总。

import lombok.Data;

        private String splitBasis;

        /** 申报金额 */
        private BigDecimal declaredAmount;

        /** 申报税额 */
        private BigDecimal declaredTaxAmount;
    }
}

## 8. 校验与异常口径（落地建议）

1. `splitBasis` 不能为空，且必须能解析税率（`13%/9%/6%`）。
2. 上下两表按税率一一对应，不允许单边存在。
3. 专票行金额由系统计算，若上游月结数据缺失则报错。
4. 普票行金额由用户填写；若整行为空则自动删除，不报错。
5. 删除普票空行后再做“上下表一一对应”校验，避免空行造成误报。
