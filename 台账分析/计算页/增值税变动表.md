## 3. 业务定位

### 3.1 核心用途一句话

- 用于按“基础条目 + 拆分依据”形成增值税变动结果，输出未开票金额、当月开票金额、以前月度开票金额和合计。

### 3.2 上游依赖简述

- 结构口径依赖 `增值税变动表基础条目配置表`。
- 金额逻辑依赖 `PL附表-2320、2355`、`增值税销项 `、`睿景景程月结数据表-报税`、`增值税进项认证清单`、`增值税变动表附表`、明细类页签（含进项/BS取数逻辑说明）。
- 本页大量规则以“说明文字”给出，单元格公式较少。

### 3.3 下游输出对象

- 输出供 `Summary `、账税差异监控和增值税相关汇总逻辑引用。
- 同时作为本期台账中增值税模块的核心展示页。

## 7. 字段取值逻辑明细（逐字段，小分段）

### 7.1 `baseItem`（基础条目）

取值来源：`增值税变动表基础条目配置表`。  
来源字段映射（已确认）：

- 基础条目：`基础条目(basicItem)`
- 条目序号：`条目序号(itemSeq)`
- 公司代码：`公司代码(companyCode)`
- 是否展示：`是否展示(isDisplay)`
- 逻辑删除：`逻辑删除(isDeleted)`

完整逻辑：

1. 读取台账生成入参：`companyCode`、`yearMonth`。  
2. 在配置表中过滤记录：`isDeleted=0` 且 `isDisplay=Y`。  
3. 再做公司范围过滤：`companyCode = 当前公司` 或 `companyCode is null`。  
4. 按 `itemSeq` 升序排序。  
5. 逐行读取 `basicItem` 写入当前行 `baseItem`。  
6. 若 `basicItem` 为空，当前配置行丢弃，不进入后续拆分与金额计算。  

### 7.2 `splitBasis`（拆分依据）

取值来源：配置表 + 业务来源 sheet。  
来源字段映射（已确认）：

- 是否拆分：`是否拆分(isSplit)`（来自 `增值税变动表基础条目配置表`）
- PL拆分依据（非2320/2355）：`拆分依据(splitBasis)`（来自 `PL附表`）
- PL拆分依据（2320/2355）：`拆分依据(splitBasis)`（来自 `PL附表-2320、2355`）

完整逻辑：

1. 每个 `baseItem` 先读取配置 `isSplit`。  
2. `isSplit != Y` 时：`splitBasis = null`，该 `baseItem` 只生成 1 行。  
3. `isSplit = Y` 时：按公司分支选择 PL 来源表并取拆分依据：
- 若当前公司属于 `2320/2355`，从 `PL附表-2320、2355.splitBasis` 取值；
- 若当前公司不属于 `2320/2355`，从 `PL附表.splitBasis` 取值。  
4. 拆分值为空的记录剔除。  
5. 拆分值去重并保持原业务顺序，作为本 `baseItem` 的展开行。  

### 7.3 `itemName`（条目）

取值来源：`baseItem` 与 `splitBasis`。  
完整逻辑：

1. 若 `splitBasis` 非空：`itemName = baseItem + "-" + splitBasis`。  
2. 若 `splitBasis` 为空：`itemName = baseItem`。  
3. 对生成结果做唯一性校验；重复时按“`itemSeq` + 拆分顺序”重建后再落表。  

### 7.4 `unbilledAmount`（未开票金额）

取值来源：本 sheet 计算。  
完整逻辑：

1. 先判断当前行 `baseItem` 是否为以下任一值：  
- `当月利润表主营业务收入`  
- `应交增值税-销项税`  
2. 仅当满足上述条件时，才进行未开票金额计算；否则该字段按规则置空，不参与本公式。  
3. 对满足条件的行，先取得本行 `totalAmount`、`currentMonthInvoicedAmount`、`previousMonthInvoicedAmount`。  
4. 使用统一公式：  
   `unbilledAmount = totalAmount - currentMonthInvoicedAmount - previousMonthInvoicedAmount`。  
5. 结果为负时按业务规则处理（保留或报错）。  

### 7.5 `currentMonthInvoicedAmount`（当月开票金额）

取值来源：`增值税销项表-按税率（征收率）统计表` + `睿景景程月结数据表-报税`（2320/2355校验）。  
术语约定（本节统一口径）：

- 文中“条目”对应当前行 `基础条目(baseItem)`。
- 当名称后带 `*`（如“当月利润表主营业务收入*”、“应交增值税-销项税*”）时，表示该基础条目为“有拆分列”的场景（`isSplit=Y`）。
- 因此，凡满足该基础条目且被拆分展开出的所有明细行（不同 `splitBasis` 行）均适用对应规则。  
  来源字段映射（已确认）：
- `增值税销项.TaxRateSummaryItem`：
  `taxRateOrLevyRate`（税率/征收率）、`invoiceStatus`（发票类型）、`blueInvoiceAmount`（实际开具发票金额）、`blueInvoiceTaxAmount`（实际开具发票税额）
- `睿景景程月结数据表-报税.MonthlyTaxSectionDTO`：
  `title`（类别税率）、`invoicedIncome`（已开票收入）、`invoicedTaxAmount`（已开票税额）

完整逻辑：

1. 统一前置口径：
- 若台账公司 = `2320/2355`，则 `splitBasis` 固定取自 `PL附表-2320、2355.splitBasis`。
- “有普票”= `splitBasis` 文本包含“普票”（contains）。
- “无普票”= `splitBasis` 文本不包含“普票”（按专票口径）。
- “拆分依据中税率”是指从 `splitBasis` 提取税率字符串（如 `专票-6% -> 6%`、`普票-9% -> 9%`）。
2. 场景A：先在【增值税销项表-按税率（征收率）统计表】做“税率/征收率字段”匹配取值（这一步不是合计行）：
- 当基础条目 = `当月利润表主营业务收入*` 且“无普票”：  
  `taxRateOrLevyRate = 由 splitBasis 提取的税率` 且 `invoiceStatus = 电子发票（增值税专用发票）`，取 `blueInvoiceAmount` 作为销项侧金额A。
- 当基础条目 = `当月利润表主营业务收入*` 且“有普票”：  
  `taxRateOrLevyRate = 由 splitBasis 提取的税率` 且 `invoiceStatus = 电子发票（普通发票）`，取 `blueInvoiceAmount` 并直接写入 `currentMonthInvoicedAmount`。
- 当基础条目 = `应交增值税-销项税*` 且“无普票”：  
  `taxRateOrLevyRate = 由 splitBasis 提取的税率` 且 `invoiceStatus = 电子发票（增值税专用发票）`，取 `blueInvoiceTaxAmount` 作为销项侧税额A。
- 当基础条目 = `应交增值税-销项税*` 且“有普票”：  
  `taxRateOrLevyRate = 由 splitBasis 提取的税率` 且 `invoiceStatus = 电子发票（普通发票）`，取 `blueInvoiceTaxAmount` 并直接写入 `currentMonthInvoicedAmount`。
3. 场景B：仅对“无普票”分支，再在【睿景景程月结数据表-报税】做 `title` 模糊匹配汇总后校验：
- 收入分支（`当月利润表主营业务收入*` 且无普票）：  
  从 `splitBasis` 提取税率（示例：`专票-6% -> 6%`），筛选所有 `title` 包含 `6%` 的记录，汇总 `invoicedIncome` 得到月结侧金额B；校验 `A == B`，一致写入 `currentMonthInvoicedAmount`，不一致报错：  
  `当月利润表主营业务收入*的当月开票金额【增值税销项表-按税率（征收率）统计表】与【睿景景程月结数据表-报税】不一致，请检查。`
- 税额分支（`应交增值税-销项税*` 且无普票）：  
  同样按 `title contains 税率` 过滤并汇总 `invoicedTaxAmount` 得到月结侧税额B；校验 `A == B`，一致写入 `currentMonthInvoicedAmount`，不一致报错：  
  `应交增值税-销项税*的当月开票金额【增值税销项表-按税率（征收率）统计表】与【睿景景程月结数据表-报税】不一致，请检查。`
4. 场景C：若台账公司 ≠ `2320/2355`，才走“合计行”逻辑：
- 当基础条目 = `当月利润表主营业务收入`：取【增值税销项表-按税率（征收率）统计表】合计行“实际开具发票金额”。
- 当基础条目 = `应交增值税-销项税`：取【增值税销项表-按税率（征收率）统计表】合计行“实际开具发票税额”。
- 其他情况：`currentMonthInvoicedAmount` 为空。  

### 7.6 `previousMonthInvoicedAmount`（以前月度开票金额）

取值来源：用户手工维护。  
完整逻辑：

1. 该字段不做自动取数。  
2. 用户录入后直接写入 `previousMonthInvoicedAmount`。  
3. 系统执行校验：数值合法性、必填条目非空、异常波动提示。  
4. 未填时按系统约定处理（通常 `null`，必要时可配置为0）。  

### 7.7 `totalAmount`（合计）

取值来源：`PL附表/PL附表-2320、2355`、`增值税进项认证清单`、`增值税变动表附表`、`进项明细`、上期台账（税务台账表-增值税变动表）。  
术语约定（本节统一口径）：

- 文中“条目”对应当前行 `基础条目(baseItem)`。
- 名称后带 `*` 表示该基础条目处于拆分场景（`isSplit=Y`），规则适用于该基础条目下所有拆分行。  
  来源字段映射（已确认）：
- `PL附表` / `PL附表-2320、2355`：`splitBasis`、`mainBusinessRevenue`、`outputTax`
- `增值税进项认证清单`：`taxAmount`（“税额合计”定义为所有行 `taxAmount` 求和）
- `增值税变动表附表`：`currentPeriodPrepaidDeduction`
- `进项明细`：`account`、`localAmount`（其中 `account = 2221010400`）
- 上期台账（税务台账表-增值税变动表）：`期末留抵进项税`行“合计金额”

完整逻辑：

1. 公共口径与字段定义（本节全部规则先按此解释）：
- 本页“合计”字段统一指当前 sheet 的 `totalAmount`。
- `sumTotalAmountByBaseItem(X)`：指本页中 `baseItem = X` 的所有行 `totalAmount` 求和（含拆分行）。
- `certifiedInputTaxTotal`：指 `SUM(增值税进项认证清单.taxAmount)`（所有行 `taxAmount` 合计）。
- `inputTransferOutTotal`：指 `SUM(进项明细.localAmount WHERE account = '2221010400')`。
- `openingRetainedInputTaxTotal`：指“上期台账（台账月份-1）中 baseItem=期末留抵进项税 的 totalAmount”；若当月为1月，取上年12月。
2. 若台账公司 = `2320/2355`：
3. 拆分依据匹配口径：
- 数据源固定为 `PL附表-2320、2355`；
- 当前行 `splitBasis` 与来源表 `splitBasis` 先精确匹配，失败再模糊包含匹配。
2. 当 `baseItem = 当月利润表主营业务收入*`：  
   `totalAmount = PL附表-2320、2355.mainBusinessRevenue`（按 `splitBasis` 匹配命中行）。

3. 当 `baseItem = 应交增值税-销项税*`：  
   `totalAmount = PL附表-2320、2355.outputTax`（按 `splitBasis` 匹配命中行）。

4. 当 `baseItem = 增值税已认证进项税`：  
   `totalAmount = certifiedInputTaxTotal`。

5. 当 `baseItem = 期初留抵进项税`：  
   `totalAmount = openingRetainedInputTaxTotal`。

6. 当 `baseItem = 期末留抵进项税`：  
   `X = sumTotalAmountByBaseItem('应交增值税-销项税') - ( sumTotalAmountByBaseItem('增值税已认证进项税') + sumTotalAmountByBaseItem('期初留抵进项税') - sumTotalAmountByBaseItem('进项转出') )`；  
   若 `X < 0`，`totalAmount = abs(X)`；若 `X > 0`，`totalAmount = 0`。

7. 当 `baseItem = 应交增值税`：  
   若 `sumTotalAmountByBaseItem('期末留抵进项税') > 0`，`totalAmount = 0`；  
   若 `sumTotalAmountByBaseItem('期末留抵进项税') < 0`，  
   `totalAmount = sumTotalAmountByBaseItem('销项税额*') + sumTotalAmountByBaseItem('应交增值税-销项税*') + sumTotalAmountByBaseItem('进项转出') - sumTotalAmountByBaseItem('增值税已认证进项税') - sumTotalAmountByBaseItem('期初留抵进项税')`。

8. 当 `baseItem = 本期应交增值税`：  
   `totalAmount = sumTotalAmountByBaseItem('应交增值税') - sumTotalAmountByBaseItem('异地预缴抵减')`。

9. 当 `baseItem = 异地预缴抵减`：  
   `totalAmount = 增值税变动表附表.currentPeriodPrepaidDeduction`。

10. 当 `baseItem = 进项转出`：  
    `totalAmount = inputTransferOutTotal`。

11. 其他情况：`totalAmount = null`。

12. 若台账公司 ≠ `2320/2355`：

13. 拆分依据匹配口径：
- 数据源优先 `PL附表`；
- 当前行 `splitBasis` 与 `PL附表.splitBasis` 先精确匹配，失败再模糊包含匹配；
- 若无 `PL附表`，按下述兜底规则取 `PL表/BS表`。
2. 当 `baseItem = 当月利润表主营业务收入`：  
   优先 `totalAmount = PL附表.mainBusinessRevenue`（按 `splitBasis` 命中行）；  
   无 `PL附表` 时按 `PL表（利润表）` 取值：  
- 定位 `项目(itemName) = 主营业务收入` 的数据行；  
- 取该行 `本期发生额(currentPeriodAmount)`；  
- 写入当前行 `totalAmount`。
3. 当 `baseItem = 应交增值税-销项税`：  
   优先 `totalAmount = PL附表.outputTax`（按 `splitBasis` 命中行）；  
   无 `PL附表` 时按 `BS表（资产负债表）` 取值：  
- 定位 `项目(itemName) = 应交税费` 的数据行；  
- 读取该行 `累计发生数(accumulatedAmount)` 与 `年初数(yearStartAmount)`；  
- 计算 `totalAmount = (accumulatedAmount - yearStartAmount) * -1`。
4. 当 `baseItem = 增值税已认证进项税`：  
   `totalAmount = certifiedInputTaxTotal`。
5. 当 `baseItem = 期初留抵进项税`：  
   `totalAmount = openingRetainedInputTaxTotal`。
6. 当 `baseItem = 期末留抵进项税`：  
   `X = sumTotalAmountByBaseItem('应交增值税-销项税') - ( sumTotalAmountByBaseItem('增值税已认证进项税') + sumTotalAmountByBaseItem('期初留抵进项税') - sumTotalAmountByBaseItem('进项转出') )`；  
   若 `X < 0`，`totalAmount = abs(X)`；若 `X > 0`，`totalAmount = 0`。
7. 当 `baseItem = 应交增值税`：  
   若 `sumTotalAmountByBaseItem('期末留抵进项税') > 0`，`totalAmount = 0`；  
   若 `sumTotalAmountByBaseItem('期末留抵进项税') < 0`，  
   `totalAmount = sumTotalAmountByBaseItem('销项税额*') + sumTotalAmountByBaseItem('应交增值税-销项税*') + sumTotalAmountByBaseItem('进项转出') - sumTotalAmountByBaseItem('增值税已认证进项税') - sumTotalAmountByBaseItem('期初留抵进项税')`。
8. 其他情况：`totalAmount = null`。  

### 7.8 全局分支与优先级规则

1. 先按公司分支：`2320/2355` 与非 `2320/2355` 使用不同校验链路。  
2. 再按条目分支：先确定 `baseItem/splitBasis`，再决定取数 sheet。  
3. 跨期条目优先读取上期台账，不得被当期明细覆盖。  
4. 手工字段（`previousMonthInvoicedAmount`）不被自动取数覆盖。  
5. 任一来源缺失时只影响其对应条目，不允许“借用其他条目来源”补值。  

### 7.9 推荐实现顺序（代码落地）

1. 生成结构字段：`baseItem -> splitBasis -> itemName`。  
2. 路由并计算金额字段：`totalAmount`、`currentMonthInvoicedAmount`、`previousMonthInvoicedAmount`。  
3. 统一计算 `unbilledAmount` 并执行校验（必填、数值、跨表一致性、公司分支一致性）。
