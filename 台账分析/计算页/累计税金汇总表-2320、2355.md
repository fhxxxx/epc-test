## 2. Sheet结构重新分析

- UsedRange：`A1:R50`
- 核心主表：`A1:P22`
- 一页多表：否（本次仅分析主汇总表）
- 冻结窗格：无
- 主表核心形态：
  - 列方向是期间（`202412` ~ `202512`）
  - 行方向是指标（账面收入、销项税额、印花税等）
  - 末列给出“本台账期”取值规则
  - 建模采用“转置建模”：每个期间列映射为一个对象，左侧固定指标映射为对象字段

### 2.1 全局生成规则（长期维护）

- 本sheet是“长期维护型累计表”，不是每期全量重建。
- 每次生成新台账时，必须先读取“上一期已生成台账”中的本sheet完整数据列。
- 生成动作固定为：在最右侧新增1列“当前台账月份”。
- 历史期间列全局约束：
  - 不修改
  - 不重算
  - 不回刷
- 仅新增的当前月份列参与本次计算与写入。
- 对应到对象模型：
  - `List<CumulativeTaxSummary23202355ColumnDTO>` 先承接上一期全部对象；
  - 再 append 一个 `period = 当前台账月份` 的新对象。

## 6. 字段取值逻辑（按对象字段顺序）

### 6.1 `period`

- 取列头期间值（如 `202412`、`202501`、...、`202512`）。
- 每一列对应 `List<CumulativeTaxSummary23202355ColumnDTO>` 的一个对象。
- 生成机制是“上一期继承 + 本期追加”：
  - 先加载上一期台账本sheet已有期间对象；
  - 校验最后一列期间为“上一台账月份”；
  - 在列表末尾新增当前月份对象（`period = 当前台账月份`）。

### 6.2 `bookIncome`

- `账面收入`：来自 `增值税变动表.totalAmount`，条件：`baseItem = 当月利润表主营业务收入*`，按行汇总。
- 本字段仅对“新增的当前月份对象”执行计算并写入。

### 6.3 `otherIncome`

- 来自 `其他科目明细.localAmount`，条件：`subjectCode = 6702000010`，求和。

### 6.4 `projectInvoicing`

- 来自 `增值税变动表.currentMonthInvoicedAmount`，条件：`baseItem = 当月利润表主营业务收入*`，按行汇总。

### 6.5 `interestInvoicing`

- 来自 `其他科目明细.localAmount`，条件：`subjectCode = 6603020011`，求和。

### 6.6 `outputTaxA`

- 来自 `增值税变动表.totalAmount`，条件：`baseItem = 应交增值税-销项税*`，按行汇总。

### 6.7 `currentInputTaxB`

- 来自 `增值税变动表.totalAmount`，条件：`baseItem = 增值税已认证进项税`。

### 6.8 `openingRetainedInputTaxC`

- 来自 `增值税变动表.totalAmount`，条件：`baseItem = 期初留抵进项税`。

### 6.9 `inputTaxTransferOutD`

- 来自 `增值税变动表.totalAmount`，条件：`baseItem = 进项转出`。

### 6.10 `remotePrepaidVatE`

- 来自 `增值税变动表.totalAmount`，条件：`baseItem = 异地预缴递减`。

### 6.11 `vatPayableAminusBminusCplusDminusE`

- 来自 `增值税变动表.totalAmount`，条件：`baseItem = 本期应交增值税`。

### 6.12 `vatAmount`

- 等于 `vatPayableAminusBminusCplusDminusE`（本sheet口径为“增值税=应纳增值税额”）。

### 6.13 `stampDuty`

- 来自 `Summary ` 中税种=印花税的 `actualTaxPayable`（实际应缴税额）。

### 6.14 `urbanConstructionTax`

- `vatAmount * 税目配置表.taxRate`（税目=城建税）。

### 6.15 `educationSurcharge`

- `vatAmount * 税目配置表.taxRate`（税目=教育费附加）。

### 6.16 `localEducationSurcharge`

- `vatAmount * 税目配置表.taxRate`（税目=地方教育费附加）。

### 6.17 `propertyTax`

- 来自 `Summary ` 中税种=房产税的 `actualTaxPayable`。

### 6.18 `urbanLandUseTax`

- 来自 `Summary ` 中税种=城镇土地使用税的 `actualTaxPayable`。

### 6.19 `corporateIncomeTax`

- 来自 `Summary ` 中税种=企业所得税合计的季度应缴口径字段。

### 6.20 `individualIncomeTax`

- 来自 `Summary ` 中税种=个税的应缴字段（若无值则为0）。

### 6.21 `disabledPersonsEmploymentSecurityFund`

- 来自 `Summary ` 中税种=残疾人保障金的 `actualTaxPayable`。

### 6.22 `totalTaxAmount`

- `vatAmount + stampDuty + urbanConstructionTax + educationSurcharge + localEducationSurcharge + propertyTax + urbanLandUseTax + corporateIncomeTax + individualIncomeTax + disabledPersonsEmploymentSecurityFund`。

### 6.23 右侧橙色列说明（非对象字段）

- 右侧橙色列是开发实现规则说明区，不属于业务数据，不入库、不进 Java DTO、也不进 JSON。
- 该区域用于指导上方各字段（`bookIncome`~`totalTaxAmount`）在“本台账期列”的取值实现逻辑。

## 7. 校验与风险

- 校验1：`合计` 行必须逐月等于 `增值税` 至 `残疾人保障金` 各行求和。
- 校验2：`城建税/教育费附加/地方教育费附加` 必须满足“增值税 * 税目配置税率”。
- 校验3：`应纳增值税额` 与 `增值税变动表(baseItem=本期应交增值税).totalAmount` 一致。
- 风险1：`Summary ` 税种命名不一致会导致税种匹配失败。
- 风险2：`增值税变动表` 基础条目口径变化会连带影响本sheet全部核心金额。
